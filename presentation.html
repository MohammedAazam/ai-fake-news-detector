1. Motivation & Problem Statement

The Core Challenge: Misinformation spreads virally before manual fact-checkers can respond. Furthermore, traditional NLP models (SVMs, LSTMs) struggle to grasp sarcasm, context, and long-range semantic dependencies.

The Cognitive Barrier: Users exhibit confirmation bias. If an AI acts as a "black box" and simply labels an article as "Fake" without providing mathematical evidence, users frequently reject the classification (the backfire effect).

The Deployment Gap: Large Language Models are computationally heavy. Forcing users to copy/paste text into third-party websites creates friction, preventing widespread adoption.

Our Proposed Solution:

  • Dual-Model Ensemble: DistilBERT for speed, RoBERTa for accuracy.
  • Explainable AI (XAI): Captum Integrated Gradients for token-level transparency.
  • Browser Native: Manifest V3 extension for zero-friction user experience.

3. Soft-Voting Ensemble Methodology

Mathematical Formulation

Rather than relying on hard majority voting, we compute the arithmetic mean of the class probabilities generated by both models. This prevents a single over-confident model from skewing the final result.

$$ P_{fake}^{Ens} = \frac{P_{fake}^{DistilBERT} + P_{fake}^{RoBERTa}}{2} $$
$$ P_{real}^{Ens} = \frac{P_{real}^{DistilBERT} + P_{real}^{RoBERTa}}{2} $$
$$ \hat{Y} = \arg\max \left(P_{fake}^{Ens}, P_{real}^{Ens}\right) $$

If \(\arg\max(P^{DB}) \neq \arg\max(P^{RB})\), the API toggles models_agree = false, triggering a UI warning.

FastAPI Implementation

# app.py - Core Ensemble Endpoint
@app.post("/predict/compare")
async def predict_compare(request: NewsRequest):
    # Execute parallel independent inferences
    res_db = run_inference(request.text, model_distilbert)
    res_rb = run_inference(request.text, model_roberta)

    # Calculate Soft-Voting Means
    avg_fake = (res_db["prob_fake"] + res_rb["prob_fake"]) / 2
    avg_real = (res_db["prob_real"] + res_rb["prob_real"]) / 2
    
    label = "Fake" if avg_fake > avg_real else "Real"
    models_agree = res_db["prediction"] == res_rb["prediction"]

    return {
        "distilbert": res_db,
        "roberta": res_rb,
        "ensemble": {
            "prediction": label,
            "confidence": max(avg_fake, avg_real),
            "models_agree": models_agree
        }
    }

4. Explainable AI (XAI) & Token Cleansing

We utilize Integrated Gradients to compute the path integral of the gradients from a zero-tensor baseline to the actual input tokens. The attribution score \(IG_i\) is approximated using a Riemann sum:

$$ IG_i \approx (x_i - x'_i) \times \frac{1}{m} \sum_{k=1}^{m} \frac{\partial F(x' + \frac{k}{m}(x-x'))}{\partial x_i} $$

Sub-Word Cleansing Heuristic

Tokenizer outputs are fragmented (e.g., [CLS], Ġshock, ##ing). Our algorithm cleans these tensors before returning JSON to ensure readable UI "Red Flags".

# Heuristic Token Filtering snippet
skip_tokens = {'[CLS]', '[SEP]', '[PAD]', '', ''}
token_importance = []

for t, s in zip(tokens, attributions):
    # Ignore special tokens and DistilBERT sub-words
    if t in skip_tokens or t.startswith('##'):
        continue
    # Clean RoBERTa BPE spacing prefixes
    clean_t = t.lstrip('Ġ') 
    if clean_t and clean_t not in skip_tokens:
        token_importance.append((clean_t, s))
        
# Return Top 5 Red Flags
token_importance.sort(key=lambda x: abs(x[1]), reverse=True)
